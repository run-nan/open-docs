plugin.yaml

The plugin configuration file describes the abilities, modules and other related information used in the plugin. The Open Platform will consume the content of the file when the plugin installation and enablement will be installed and enabled.

File path and structure:

/config/plugin.yaml
service:
  app_id: '********'
  version: 1.0.0
  name: MyPluginName
  description: MyPluginDescription
  language: nodejs
  language_version: 16.13.0
  host_version: 1.0.0
  host_sub_version: 1.0.0
  min_system_version: 3.5.0
  contact: MyContactInformation
  config:
    - key: MyCustomConfigKey
      type: Input
abilities:
  - id: Simple-login-SXgh
    name: SimpleLoginVerification
    abilityType: SimpleAuth
    function:
      validateFunc: SimpleAuthValidate
apis:
  - type: addition
    methods:
      - GET
    url: /hello
    function: HelloWorld
modules:
  - id: ones-global-banner-GXvd
    title: example banner title
    moduleType: ones:global:banner
    entry: modules/ones-global-banner-GXvd/index.html

serviceâ€‹

The basic information configuration of the plugin, the parameters are as follows:

app_idâ€‹

  • Type: string

The unique ID of the plugin is automatically generated by the cli. You are not advised to change it.

The two plugins, even if the name is the same, if the app_id is different, they will be recognized as different plugins; on the contrary, if the app_id is consistent, whether the name is the same, they will be recognized as the same plugin.

If a plugin needs to be upgraded, it is necessary to ensure that the app_id of the new and old versions must be consistent, otherwise the open platform will be deemed to be installed as a new plugin.

versionâ€‹

  • Type: string

The plugin version, it is recommended to follow Semantic Versioning 2.0.0.

nameâ€‹

  • Type: string

The plugin name will be displayed on the Plugin Management and Details page.

descriptionâ€‹

  • Type: string

The plugin description will be displayed on the Plugin Management and Details page.

scopeâ€‹

  • Type: 1 | 2

The scope of the plugin.

TypeDescription
1Team-level plugin, can only be installed under the "Configuration Center" => "Plugin Management", and can only take effect on the current team
2Organizational-level plugin can only be installed under "Organization Management" => "Plugin Management", and take effect on all teams under the organization

languageâ€‹

  • Type: 'nodejs'

The development language of the plugin currently only supports Node.js, which generally does not need to be modified.

host_versionâ€‹

  • Type: string

The lowest version number of the Open Platform is generally not modified.

When the plugin is enabled, it will run on the HOST of the highest digit patch version, which is similar to the ~version of npm.

Example:

If your current host_version version is 1.0.4.

The server has the following HOST versions:

  • 1.0.3
  • 1.0.4
  • 1.0.5
  • 1.1.0
  • 2.0.0

Then your plugin will run on the HOST of the 1.0.5 version.

contactâ€‹

  • Type: string
  • Max: 255

The contact information displayed by the plugin log (support multi-line text).

Example:

/config/plugin.yaml
service:
  contact: |-
    email: support@ones.com
    Call: 400-188-1518

Plugin custom standard permission point configuration item array, you can use this configuration quickly build your own plugin permission system, For more information, please see the Plugin Custom Permission document.

abilitiesâ€‹

The plugin capacity configuration item array, the configuration parameters of a single ability are as follows:

idâ€‹

  • Type: string

A unique identifier of the plugin ability.

nameâ€‹

  • Type: string

The ability name of the plugin.

labelâ€‹

  • Type: string
  • Reserved Word

The ability alias of the plugin.

relateModuleâ€‹

  • Type: string

The Plugin Module ID used with the current ability binding is configured according to specific ability requirements, and generally no modification is required.

versionâ€‹

  • Type: string

Plugin ability version when the cli adding ability, it is automatically generated by the cli, and generally does not need to be modified.

abilityTypeâ€‹

  • Type: string

Plugin ability type, it is automatically generated by the cli, and generally does not need to be modified.

functionâ€‹

  • Type: Record<string, string>

The method name of the ability to call the plugin ability is automatically generated by the cli. If the same ability is added many times, you need to manually modify the specified new method name.

configâ€‹

  • Type: AbilityConfig[]

Plugin ability configuration item, it is automatically generated by the cli

AbilityConfig Params:

KeyDescriptionTypeRequiredDefault
keyUnique identificationstringY
fieldTypeControl typeFieldTypeY
requiredIs this requiredbooleanNfalse
labelAliasstringNkey
valueDefaultstringN
showWhether the plugin detail page shows this configurationbooleanNtrue
propsConfiguration item UI input control propsobjectN

FieldType Type:

  • 'Input'ï¼šSingle-line text
  • 'TextArea'ï¼šMulti-line text
  • 'Password'ï¼šPassword input
  • 'Number'ï¼šNumber input
  • 'Select'ï¼šSingle select
  • 'MultiSelect'ï¼šMultiple select
  • 'DatePicker'ï¼šDate picker
  • 'DateTimePicker'ï¼šDate-time picker
  • 'TimePicker'ï¼šTime picker
  • 'Switch'ï¼šSwitch

props Parameter:

The props of the UI input control for the configuration item. See the ONES Design documentation for the specific parameters in conjunction with FieldType.

Example:

When your FieldType is a text input box, you can control the placeholder of the input box, etc. via props.

When your FieldType is a select box, you can control the options of the select box, etc. via props.

caution

Only some parameters are supported (and limited by yaml to string, number or null type parameters), it is recommended that you do not pass parameters that affect functionality, we do not guarantee that the configuration form will function properly after passing them.

apisâ€‹

The interface configuration item of the plugin array, the configuration parameters of the single interface are as follows:

typeâ€‹

  • Type: 'external' ï½œ 'addition' | 'replace' | 'prefix' | 'suffix'

Tnterface type, For more information, please see the Interface Registration document.

  • 'external'ï¼šThe team level new interface (will check the login, and the URL needs to bring the team/{teamUUID} prefix)
  • 'addition'ï¼šThe plugin level new interface
  • 'replace'ï¼šReplace the ONES standard interface (to be abandoned)
  • 'prefix'ï¼šONES standard interface prefix interception (to be abandoned)
  • 'suffix'ï¼šONES standard interface suffix interception (to be abandoned)

methodsâ€‹

  • Type: ('GET' | 'POST')[]

The interface support request method can set multiple types at the same time.

urlâ€‹

  • Type: string
  • Example: /hello

Interface access address.

functionâ€‹

  • Type: string

The call entry method name during interface access (this method must be export to be called by the Open Platform).

scopeâ€‹

  • Type: 'project' | 'wiki'
  • Default (Inculde undefined): 'project'

The current interface is to modify the ONES standard interface (that is, type is 'replace' | 'prefix' | 'suffix'), you need to declare that the interface you currently modified project or wiki interface.

note

If it is a new interface with a plugin, this field is meaningless.

modulesâ€‹

The module configuration item of the plugin array, the single module configuration parameter is as follows:

idâ€‹

  • Type: string

The unique module ID within the plugin, it is automatically generated by the cli, and generally does not need to be modified.

titleâ€‹

  • Type: string

Module title.

descriptionâ€‹

  • Type: string

Module description.

iconâ€‹

  • Type: string
  • Example: ./logo.svg

Module icon addressï¼Œthe Open Platform will query svg icon resources through the relative path in the plugin /web/public directory.

linkâ€‹

  • Type: string

The jump link to a module, which is mutually exclusive with entry.

entryâ€‹

  • Type: string

The module entry file path, and link mutually exclusive, it is automatically generated by the cli, and generally does not need to be modified.

disabledâ€‹

  • Type: boolean

Module Specifies whether to disable modules of standard ONES.

caution

The new type of slot does not support the setting of disabled. If you need to remove the added plugin module, you can comment or delete the corresponding module under the modules.

manualâ€‹

  • Type: boolean

Whether the module is triggered by the user's specific operation and configuration according to specific capabilities, generally no modification is required.

preloadâ€‹

  • Type: boolean

Whether the module needs to be pre-loaded, this is usually used in conjunction with the manual and configured according to specific capabilities and needs. Generally, no modification is required.

moduleTypeâ€‹

  • Type: string

Module Type, For optional values, see Slot Module List, it is automatically generated by the cli, and generally does not need to be modified.

permissionâ€‹

  • Type: string

Permission identity, which works with the plugin custom standard permission point configuration or the plugin custom permission point ability, For more information, please see here.

Example:

With the custom standard permission point configuration to control the current module effectiveness permissions, administrators can control the "Effective Permission" permissions on the plugin permissions configuration page to control the effective user of the "ones-global-banner-GXvd" module.scope.

/config/plugin.yaml
service:
  permission:
    - name: Effective Permission
      field: my-example-effective-permission-key
      desc: This is an example of custom effective permission
modules:
  - id: ones-global-banner-GXvd
    title: example banner title
    moduleType: ones:global:banner
    entry: modules/ones-global-banner-GXvd/index.html
    permisson: my-example-effective-permission-key

needPermissionContextâ€‹

  • Type: boolean
  • ONES requirements:: v3.13.59+

Whether permission validation depends on page context. When set to true, module permissions will be revalidated when the page context changes. Otherwise, module permissions will only be validated once upon the initial page entry. This feature needs to be used in conjunction with the custom permission ability of the plugin. For more details, refer to Custom Permission.

enableDelayDestroyâ€‹

  • Type: boolean

  • ONES requirements: v3.13.58+

Whether the module should be delayed to destroy, which only works with the plugin that the manual configuration is true.

caution

If you need to delay destroying the module, your plugin is still activated in ONES for some time. The plugin maybe still can receive the message from the ONES or send the message to the ONES(For example, the op-store and the op-event). You should uninstall the plugin's root component to cancel this actions except for the provider of @ones-design which can be used to toast a message, maybe you need the OPProvider to do this.

enableMemoryRouterâ€‹

  • Type: boolean

  • ONES requirements: v3.13.58+

In this router mode, you can use other third-party routers in the plugin.

This router mode creates a new location and history for the plugin. The URL that the plugin gets is the entry of the module and is saved in the browser address bar as a query argument.

caution

Before using this mode, you can get the real URL in the browser address bar by window.location.href. After opening this mode, you can't depend on the data from that URL anymore.

If you want to use the BrowserRouter in the react-router-dom, you need to set the basename which can be gotten by the window.__op_router_basename__.

For example:

import { BrowserRouter } from 'react-router-dom'
ReactDOM.render(
  <ConfigProvider>
    <BrowserRouter basename={window.__op_router_basename__}>
      <App />
    </BrowserRouter>
  </ConfigProvider>,
  document.getElementById('ones-mf-root')
)

Or you can use the HashRouter directly.

import { HashRouter } from 'react-router-dom'
ReactDOM.render(
  <ConfigProvider>
    <HashRouter>
      <App />
    </HashRouter>
  </ConfigProvider>,
  document.getElementById('ones-mf-root')
)
caution

When you use the Link in the react component to render the tag <a /> and allow the user to open the link in the new tab by right-clicking, the browser will open a wrong page, for this case, the tag <a href=""/> is not recommended. If you need to jump to the ONES and know the absolute path, you can use the tag <a href="absolutePath"/>.
