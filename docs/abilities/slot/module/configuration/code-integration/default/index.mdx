# Default group

## Requirements

|   ONES   |
| :------: |
| v3.14.0+ |

## Overview

<Image img={require('./images/default.png')} />

We have provided slots for "Configuration" - "Code Integration settings" - "[Default group](../../index.mdx#slot-hierarchy)" to customize the page module for the application.

The slots currently supported are as follows:

- [ones:settings:code_integration:default:new](#onessettingspipeline_integrationdefaultnew)
- [ones:settings:code_integration:default:repo](#onessettingspipeline_integrationdefaultrepo)
- [ones:settings:pipeline:default:repo (deprecated)](#onessettingspipelinedefaultrepo-deprecated)

## Slots

#### ones:settings:code_integration:default:new

**Module path：** Configuration / Code Integration settings / [Default group](../../index.mdx#slot-hierarchy) / new module

**Number limit：** Declare a maximum of **10** slots in each plugin, and up to **100** slots in the system.

#### ones:settings:code_integration:default:repo

**Module path：** Configuration / Code Integration settings / [Default group](../../index.mdx#slot-hierarchy) / Link repositories

**Number limit：** Declare a maximum of **1** slots in each plugin, and up to **1** slots in the system.

#### ones:settings:pipeline:default:repo (deprecated)

**ONES Requirement:** v3.11.15+

**Module path：** Configuration / Pipeline settings / [Default group](../../index.mdx#slot-hierarchy) / Manage repositories

**Number limit：** Declare a maximum of **1** slots in each plugin, and up to **1** slots in the system.

## Usage

```yaml
modules:
  - id: ones-settings-code_integration-default-repo-YA0a
    title: Link repositories
    moduleType: ones:settings:code_integration:default:repo
    entry: modules/ones-settings-code_integration-default-repo-YA0a/index.html
```

**Supported config options：**

- [disabled](../../../../../../reference/config/plugin.yaml#disabled)
