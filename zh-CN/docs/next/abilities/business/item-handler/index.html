Item 处理器

要求​

ONES
v3.11.17+

概述​

当你希望在各个模块发生变更时做一些自定义的处理和校验，可以使用 Item 处理器 来实现需求。该能力能够在系统产生变更事件时，对事件做前置处理和后置通知。当安装了使用该能力的插件后，变更事件会在实际生效之前先由插件进行前置处理，插件可根据需求对本次事件采取忽略、拒绝和修改等操作；而在事件处理完成之后，插件会收到一个通知，此时可进行一些后置的处理。

能力表现​

使用 item 处理器能力对预估工时做校验，修改预估工时不合法时报错

使用​

当 ONES 启用了使用 item 处理器能力的插件后，模块变更事件的执行流程如下图所示：

第一步：添加能力​

使用npx op add ability 添加item-handler类型的能力，新增backend/src/itemHandler.ts文件并在插件配置文件中生成能力配置。

第二步：修改配置​

在生成的能力配置中，我们只需关注itemType和action字段的值。

  • itemType

    该能力关注的模块类型，使用小驼峰书写规范。例如：project,remainingHour,activityRelease

  • action

    该能力关注的变更事件类型，可选值为add、update和delete

示例：关注manhour,remainingHour和task模块的update和add类型的变更事件。

config/plugin.yaml
abilities:
  - id: g0chW1Oh
    name: Item处理器
    version: 1.0.0
    abilityType: ItemHandler
    function:
      itemPreActionFunc: itemPreAction
      itemActionDoneFunc: itemActionDone
    config:
      - key: itemType
        value: remainingHour,task,manhour #小驼峰规范
        fieldType: Input
        label: Item类型
        show: true
      - key: action
        value: add,update
        fieldType: Input
        label: 操作
        show: true

第三步：添加前置处理​

backend/src/item-handler.ts文件中会自动生成itemPreAction函数，该函数是前置处理函数，在事件生效前会先调用该函数，支持修改本次变更内容，并根据返回内容采取不同的策略。

  • 入参介绍

    在request的body中包含以下内容

    参数类型必填说明
    user_uuidstring是用户 uuid
    item_typestring是模块类型
    actionstring是操作类型: add/update/delete
    langstring是系统语言
    graphql_queryGraphQL.Query否本次变更事件的 GraphQL 数据
    args根据事件触发方式的不同，会有以下几种类型:
    新增项目：add_project
    预估工时：estimated_manhour
    剩余工时：remain_manhour
    工作项：TaskEvent
    GraphQL 方式：map[string]interface    		是变更事件的参数

  • 返回内容

    itemPreAction函数返回内容的结构体如下所示，根据is_follow、is_reject字段的值采取不同策略：

    • 忽略：设置返回的is_follow = false，表示插件不做额外处理，更新事件按正常流程执行。
    • 拒绝：设置返回的is_follow = true && is_reject = true，表示不执行此次更新事件，直接结束。
    • 接受：设置返回的is_follow = true && is_reject = false，表示接受此次事件，但不修改此次更新内容，只在事件完成之后通知插件。
    • 修改：设置返回的is_follow = true && is_reject = false ，修改入参args中的参数值。表示接受此次事件并对此次的更新内容进行修改，事件完成之后通知插件。
    注意

    即使插件逻辑没有修改本次变更事件的内容也需要将原来入参中args对象返回，否则可能导致本次变更事件失败。

    返回结构体：

    type itemPreResponse = {
      statusCode: number
      body: {
        code: number
        body: {
          is_follow: boolean
          is_reject: boolean
          reject_reason: string
          args: any
          other_data: string
        }
      }
    }
    参数类型必填说明
    is_followstring是是否关注
    is_rejectstring是是否拒绝
    reject_reasonstring否拒绝原因
    args根据事件触发方式的不同，会有以下几种类型:
    新增项目：add_project
    预估工时：estimated_manhour
    剩余工时：remain_manhour
    工作项：TaskEvent
    GraphQL 方式：map[string]interface    		是变更事件的参数
    other_dataany否其他数据（该数据会传递到 itemActionDone）

  • 示例：预估工时必须是 8 的倍数，剩余工时修改上限为 72，超过上限只保留 72 个小时。

    backend/src/item-handler.ts
    export async function itemPreAction(request: PluginRequest): Promise<PluginResponse> {
      //The default multiple of the value, for example, 10 defaults to 1000000
      const multiple = 100000
      const body = request?.body as any
      const args = body.args
      const itemType = body.item_type
      const action = body.action
      const isFollow = true
      let isReject = false
      let reject_reason = ''
      if (itemType == 'remainingHour' && action == 'update') {
        const hours = args.hours
        if (hours > 72 * multiple) {
          args.hours = 72 * multiple
        }
      } else if (itemType == 'manhour' && action == 'update') {
        // 工时item 更新
        const type = args.type
        if (type == 'estimated') {
          // 预估工时
          const hours = args.hours
          if (hours % (8 * multiple) != 0) {
            isReject = true
            reject_reason = 'Estimated hours must be a multiple of 8'
          }
        }
      }
      return {
        statusCode: 200,
        body: {
          code: 200,
          body: {
            is_follow: isFollow,
            is_reject: isReject,
            reject_reason: reject_reason,
            args: args,
            other_data: '',
          },
        },
      }

  • 特殊说明：如果想页面弹一个自定义的warning toast 可以通过以下方式进行设置

    backend/src/item-handler.ts
    export async function itemPreAction(request: PluginRequest): Promise<PluginResponse> {
      return {
        body: {
          code: 400,
          reason: 'Test warning error',
          type: 'warning',
          model: 'plugin.xxx',
        },
      }
    }

第四步：添加后置处理​

backend/src/item-handler.ts文件中会自动生成itemActionDone函数，该函数是后置处理，在事件生效之后会先调用该函数。

  • 入参介绍

    参数类型必填说明
    user_uuidstring是用户的uuid
    item_typestring是模块类型
    actionstring是操作类型: add/update/delete
    langstring是系统语言
    graphql_queryGraphQL.Query否本次变更事件的 GraphQL 数据
    args根据事件触发方式的不同，会有以下几种类型:
    新增项目：add_project
    预估工时：estimated_manhour
    剩余工时：remain_manhour
    工作项：TaskEvent
    GraphQL 方式：map[string]interface    		是变更事件的参数
    other_dataany否其他数据（来自 itemPreAction）

  • 示例

    backend/src/item-handler.ts
    export async function itemActionDone(request: PluginRequest): Promise<PluginResponse> {
      const body = request?.body as any
      Logger.info('itemActionDone', JSON.stringify(body, undefined, 2))

      return {
        statusCode: 200,

          code: 200,
          body: {},
        },
      }
    }
