对接三方系统

要求​

ONES
v3.6.0+

概述​

该能力允许插件集成某个第三方系统(包括钉钉，谷歌，或者其他第三方系统)，插件开发者可以根据自身需要，实现以下一种或多种功能：

  1. 第三方登录：用户通过登录第三方系统，达到登录 ONES 的功能。
  2. 消息推送：ONES 的消息通知可以向第三方系统发送。
  3. 组织人员同步：把第三方系统的部门信息，人员信息同步到 ONES 中。
  • 第三方登录

在原本的登录界面下，有一个新增的其他登录方式，点击该图标即可通过第三方系统登录到 ONES

  • 消息推送

在安装实现了该能力的插件后，在各通知方式下，可以选择是否将消息推送到第三方系统

以工作项通知为例，在通知选项中多出插件的名称，选中其即可通知第三方系统

  • 组织人员同步

在配置中心->账号与成员->成员管理中，可以看到由第三方系统同步到 ONES 的用户，并且标明了信息来源的插件

使用​

通过 OP 工具添加该能力，然后实现各功能点对应的方法，最后安装启动插件，并且在 ONES 中集成第三方系统。

第一步: 通过 OP 工具添加能力​

在插件根目录使用命令行输入 OP 命令添加能力（该指令需要 OP 工具版本 1.0 及以上）

npx op add ability

选中并添加account@1.0.0

OP 工具会为插件添加如下内容

  • 插件配置文件config/plugin.yaml中 abilities 字段新增相关段落。
config/plugin.yaml
    service:
    ...
    ...
    abilities:
    - id: ...
        name: account
        version: 1.0.0
        abilityType: account
        function:
        sendMessage: SendMessage # 发送消息
        createLoginUrl: CreateLoginUrl # 创建重定向登录URL
        doExchangeUser: DoExchangeUser # 确认登录用户信息
        doPullData: DoPullData # 同步第三方系统用户数据
        config:
        # 是否实现了登录功能
        - key: canLogin
            value: true
            show: false
        # 是否实现了同步功能
        - key: canSync
            value: true
            show: false
        # 是否实现了发送消息的功能
        - key: canMessage
            value: true
            show: false
        # 第三方登录的logo地址，写的是文件名，要求该文件放在前端资源根目录下，web/dist/
        - key: logo
            value: logo.svg
            show: false
        # 第三方登录名称文案
        - key: title
            value: title
            show: false
        # 第三方描述文案
        - key: desc
            value: desc
            show: false
        # 第三方描述文案
        - key: detailTip
            value: detailTip
            show: false
        # 第三方配置提醒文案
        - key: configTip
            value: configTip
            show: false
  • OP 工具在插件中新增文件backend/src/account.ts

第二步: 实现该能力各功能点对应的方法​

实现第三方登录​

实现第三方登录 ONES 功能，只需实现CreateLoginUrl方法即可。而如果希望将第三方系统的用户信息（包括姓名，邮箱等）同步到 ONES ，并自动创建 ONES 账号，则还需要完成DoExchangeUser方法。

CreateLoginUrl需要返回第三方系统登录时调用的 url，在该方法的 request 中包含了重定向到 ONES 的 URL，插件开发者可以将其拼接入需要返回的 respUrl 中。

//在url中填入第三方系统登录时调用的url
export async function CreateLoginUrl(request: PluginRequest): Promise<PluginResponse> {
  //调用完第三方登录之后，会把返回的信息重定向到该地址，有时需要使用该地址拼接respUrl
  //@ts-ignore
  const redirectUrl: string = request?.body.redirect_url
  const response: CreateLoginUrlResponse = {
    url: '',
  }
  return CreateLoginUrlRespData(200, '200', '', '', '', response)
}

DoExchangeUser方法中，插件开发者需要返回用户的信息数据，从而在 ONES 进行用户的账号创建以及信息同步。在该方法的 request 中，有关键的 code 信息，插件开发者可以通过该 code 去第三方系统获取用户信息，然后在DoExchangeUser方法中返回用户信息。

export async function DoExchangeUser(request: PluginRequest): Promise<PluginResponse> {
  let code1 = ''
  if (typeof request?.body == 'object' && !Array.isArray(request?.body)) {
    const authInfo = request.body.auth_info
    const obj = JSON.parse(authInfo)
    //获取登录后的code，通过该code去第三方系统获取用户信息
    code1 = obj.code
  }
  //返回信息resp的格式
  const resp: DoExchangeUserResponse = {
    third_party_id: '',
    name: '',
    title: '',
    avatar: '',
    email: '',
    phone: '',
  }
  //通过调用第三方API，获取用户信息并组合数据到resp中
  return DoExchangeUserRespData(200, '', '', '', '', resp)
}

实现组织人员同步​

实现第三方的组织人员同步到 ONES 的功能只需要实现DoPullData方法即可，插件开发者需要在该方法中返回第三方部门和人员的信息，从而在 ONES 中进行部门和人员同步。

插件开发者需要先去第三方系统获取部门信息和人员信息，然后将数据重构成DoPullData方法中指定的返回结构。

export async function DoPullData(request: PluginRequest): Promise<PluginResponse> {
  const resp: DoPullDataResponse = {
    departments: [],
    users: [],
  }

  // 假设只有一个根部门，部门信息必须有一个根部门，根部门id为 "-1"
  const rootDepartment = {
    third_party_id: '-1',
    name: 'ONES',
    parent_id: '',
    next_id: '',
  }

  const departments: DepartmentInfo[] = []
  departments[0] = rootDepartment

  // 所有用户
  const userInfos: UserInfo[] = []
  userInfos.push({
    third_party_id: '1',
    name: 'test',
    email: 'test@qq.com',
    title: '职位',
    department_ids: ['-1'],
    company: 'ONES',
  })

  resp.departments = departments
  resp.users = userInfos

  return DoPullDataRespData(200, '', '', '', '', resp)
}

实现消息推送​

实现消息推送至第三方系统功能，需要插件开发者在SendMessage方法中实现自己的发送逻辑，每次 ONES 需要发送消息至第三方系统时，都会调用SendMessage方法，而消息的确切内容都会放至SendMessage方法的 request 中。

// 发送消息

第三步: 在 ONES 中进行以下操作对接第三方系统​

首先在配置中心->插件管理->插件管理->账号同步、登录与通知中安装并启用实现了该能力的插件。

然后在配置中心->账号与成员->第三方集成中点击"添加第三方集成"，选中插件进行添加。

添加成功后即可启用需要的功能点。
