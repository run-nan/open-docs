仪表盘卡片

要求​

ONES@ones-op/store@ones-op/bridge
v3.13.50+v0.10.0+v0.6.0+

概述​

开放平台提供自定义卡片的能力，支持在仪表盘中自定义一个全新的卡片类型。在当前提供的卡片类型无法满足用户需求时，可以通过自定义卡片的方式来实现。

目前用户可以自定义的内容包括：

  1. 卡片的名称、描述、图标和尺寸
  2. 卡片的配置、菜单
  3. 卡片展示的内容

目前支持添加自定义卡片类型的位置包括：

  1. 我的工作台-仪表盘
  2. 项目概览

使用​

添加模块​

  1. 执行npx op add module命令，添加ones:dashboard:card:new类型的模块
  2. 执行npx op add sub-module 命令，在刚刚添加的ones:dashboard:card:new 模块下添加子模块
  3. 在 /config/plugin.yaml 中对模块进行手动配置

下面是示例配置，一个自定义卡片模块中包含两个子模块

  • 组件卡片：scene 为 render，用来渲染自定义卡片内容，必须添加
  • 配置弹窗：scene 为 setting ，用来渲染自定义卡片配置弹窗的表单内容，如果组件不支持配置，可以不添加
plugin.yaml
service:
  app_id: cgtJVhCB
  ...
modules:
  - id: ones-dashboard-card-new-jIfq
    title: Custom Card                                  # 卡片名称
    description: Custom Card Description                       # 卡片描述
    defaultCardName: Default Card Name                  # 添加卡片时的默认名称
    icon: custom-card.svg                               # 卡片图标
    defaultLayout:                                      # 卡片的默认布局
      h: 4                                              # 高度
      w: 5                                              # 宽度
    layout:
      minH: 3                                           # 最小高度
      minW: 3                                           # 最小宽度
      maxH: 6                                           # 最大高度
      maxW: 6                                           # 最大宽度
    objectType:                                         # 卡片所属的仪表盘类型
      - project_component                               # 项目概览
      - dashboard                                       # 我的工作台
    moduleType: ones:dashboard:card:new
    modules:
      - id: ones-dashboard-card-new-C9Hs
        title: render
        scene: render
        entry: >-
          modules/ones-dashboard-card-new-jIfv/ones-dashboard-card-new-C9Hv/index.html
      - id: ones-dashboard-card-new-kjke
        scene: setting
        title: setting
        entry: >-
          modules/ones-dashboard-card-new-jIfv/ones-dashboard-card-new-kjkK/index.html

支持的配置项​

配置项说明
title自定义卡片类型的名称
description自定义卡片的描述
defaultCardName自定义卡片的默认名称
icon自定义卡片图标 url
defaultLayout自定义卡片的默认尺寸，默认尺寸需要在限制尺寸之内。（设置尺寸大小至少为 2）
layout自定义卡片的最小最大尺寸限制，不配置时默认不限制。（设置尺寸大小至少为 2）
objectType自定义卡片可应用的仪表盘类型，目前支持 dashboard（仪表盘）和 project_component（项目概览）

添加能力​

  1. 执行npx op add ability命令，添加custom-card@1.0.0类型的能力
  2. 在 /config/plugin.yaml 中对能力进行手动配置，将能力关联上模块
plugin.yaml
service:
  app_id: cgtJVhCB
  ...
modules:
  - id: ones-dashboard-card-new-jIfq
    title: Custom Card                                  # 卡片名称
    description: Custom Card Description                       # 卡片描述
    defaultCardName: Default Card Name                  # 添加卡片时的默认名称
    icon: custom-card.svg                               # 卡片图标
    defaultLayout:                                      # 卡片的默认布局
      h: 4                                              # 高度
      w: 5                                              # 宽度
    layout:
      minH: 3                                           # 最小高度
      minW: 3                                           # 最小宽度
      maxH: 6                                           # 最大高度
      maxW: 6                                           # 最大宽度
    objectType:                                         # 卡片所属的仪表盘类型
      - project_component                               # 项目概览
      - dashboard                                       # 我的工作台
    moduleType: ones:dashboard:card:new
    modules:
      - id: ones-dashboard-card-new-C9Hs
        title: render
        scene: render
        entry: >-
          modules/ones-dashboard-card-new-jIfv/ones-dashboard-card-new-C9Hv/index.html
      - id: ones-dashboard-card-new-kjke
        scene: setting
        title: setting
        entry: >-
          modules/ones-dashboard-card-new-jIfv/ones-dashboard-card-new-kjkK/index.html
abilities:
  - id: bh1Fhy76
    name: custom-card
    version: 1.0.0
    abilityType: CustomCard
    function:
      cardDataFunc: getCardData                         # 获取卡片数据的函数名
    relateModule:
      cardModule: ones-dashboard-card-new-jIfq          # 需要手动填写，必须填写moduleType为ones-dashboard-card-new的模块ID

编写处理函数​

在添加能力之后，会生成/backend/src/custom-card.js文件，其中包含一个获取自定义卡片数据的函数，需要在其中编写卡片数据的获取逻辑。

示例函数如下：

import { Logger } from '@ones-op/node-logger'
import type { PluginRequest, PluginResponse } from '@ones-op/node-types'

interface CardRequest {
  object_type: string
  language: string
  project_uuid: string
  card_custom_config: any
}

export async function getCardData(request: PluginRequest<CardRequest>): Promise<PluginResponse> {
  const cardRequest = request.body
  Logger.info(cardRequest)
  const respData: any = {
    text: cardRequest.card_custom_config.text,
  }
  return {
    body: {
      code: 200,
      body: respData,
    },
  }
}

入参介绍​

参数说明类型
object_type卡片所属的仪表盘类型，目前支持的有dashboard和project_componentstring
language本次请求的前端语言string
project_uuid项目唯一标识符,在卡片类型为project_component时存在string
card_custom_config卡片自定义配置，json 格式any

返回格式​

{
    body: {
      code: 200,   //本次请求的响应码，非200为请求错误
      body: cardData,    // 卡片数据，会直接返回给前端
    }
}

自定义卡片内容​

在scene为render的子模块中，通过useDashboardCardInfo获取卡片数据，进行卡片内容的渲染。

import { ConfigProvider } from '@ones-design/core'
import { useDashboardCardInfo } from '@ones-op/store'
import React from 'react'
import ReactDOM from 'react-dom'

import './index.css'

const Card = () => {
  const dashboardCardInfo = useDashboardCardInfo<{ text: string }>()
  const { text } = dashboardCardInfo.cardData || {}
  return <h2>{text}</h2>
}

ReactDOM.render(
  <ConfigProvider>
    <Card />
  </ConfigProvider>,
  document.getElementById('ones-mf-root'),
)

自定义卡片的菜单​

在scene为render的子模块中，通过useCustomMenus挂载自定义菜单。

import { ConfigProvider } from '@ones-design/core'
import { dashboardCard } from '@ones-op/bridge'
import { useDashboardCardInfo } from '@ones-op/store'
import React from 'react'
import ReactDOM from 'react-dom'

import './index.css'

const { useCustomMenus } = dashboardCard

const Card = () => {
  const dashboardCardInfo = useDashboardCardInfo<{ text: string }>()
  const { text } = dashboardCardInfo.cardData || {}
  const customMenus = React.useMemo(
    () => [
      {
        label: 'Show the text',
        key: 'showText',
        onClick: () => {
          alert(text)
        },
      },
    ],
    [text],
  )
  useCustomMenus(customMenus)
  return <h2>{text}</h2>
}

ReactDOM.render(
  <ConfigProvider>
    <Card />
  </ConfigProvider>,
  document.getElementById('ones-mf-root'),
)

自定义卡片配置弹窗​

在scene为setting的子模块中，通过useDashboardCardConfigInfo获取卡片上下文， 通过useOnSubmitConfig 监听提交配置事件，进行卡片配置弹窗的渲染。

import { ConfigProvider, Form, Input } from '@ones-design/core'
import { dashboardCard } from '@ones-op/bridge'
import { useDashboardCardConfigInfo } from '@ones-op/store'
import React from 'react'
import ReactDOM from 'react-dom'

import './index.css'

const { useOnSubmitConfig } = dashboardCard

const Setting = () => {
  const { cardConfig } = useDashboardCardConfigInfo<{
    text: string
  }>()
  const [form] = Form.useForm<typeof cardConfig>()
  const onSubmitConfig = React.useCallback(
    async (submit) => {
      try {
        await form.validateFields()
      } catch (e) {
        return
      }
      submit(form.getFieldsValue())
    },
    [form],
  )
  useOnSubmitConfig(onSubmitConfig)
  return (
    <Form initialValues={cardConfig} form={form}>
      <Form.Item
        name={['text']}
        label="content"
        rules={[
          {
            required: true,
          },
        ]}
      >
        <Input />
      </Form.Item>
    </Form>
  )
}

ReactDOM.render(
  <ConfigProvider>
    <Setting />
  </ConfigProvider>,
  document.getElementById('ones-mf-root'),
)

常见问题​

自定义卡片如何实现多语言？​

  • 可配置部分实现多语言，如卡片名称、卡片描述、卡片默认名称等，可参考插件多语言支持进行实现
  • 卡片展示内容多语言，需要在插件开发者自己实现，处理函数的入参中提供了本次请求的语言。
