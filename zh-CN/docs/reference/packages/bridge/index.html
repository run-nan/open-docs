@ones-op/bridge

我们提供了一套工具库，可以帮助你调用 ONES 系统内提供的一些工具方法

安装​

在插件 /web 目录，通过以下命令安装此包：

  • npm
  • Yarn
  • pnpm
npm install @ones-op/bridge
yarn add @ones-op/bridge
pnpm add @ones-op/bridge

使用​

import { modal } from '@ones-op/bridge'

modal.createTask()

APIs​

modal​

通过 modal 暴露的方法，你可以唤起 ONES 系统内的一系列弹窗

createTask​

唤起新建工作项的弹窗

要求​
ONES@ones-op/bridge
v3.13.14v0.2.0
参数​
参数说明类型必填默认值
modalProps弹窗的配置CreateTaskModalProps否{}
类型​
interface TaskCreatedResult {
  /** 创建的工作项的ID */
  taskUUID: string
  /** 跳转到工作项详情的方法 */
  queryToTask: () => void
  /** 创建的工作项的详情 */
  task: Record<string, any>
}

interface CreateTaskModalProps {
  /** 弹窗标题，如果为空，则根据选择的工作项类型自动设置 */
  title?: string
  /** 表单默认值 */
  defaultValue?: {
    /** 工作项标题 */
    summary?: string
    /** 所属项目 */
    projectUUID?: string
    /** 工作项类型 */
    issueTypeUUID?: string
    /** 负责人 */
    assign?: string
    /** 任务描述 */
    descRich?: string
    /** 所属迭代 */
    sprintUUID?: string
  }
  /** 禁止更改的表单项 */
  disabledFields?: {
    /** 工作项标题 */
    summary?: boolean
    /** 所属项目 */
    projectUUID?: boolean
    /** 工作项类型 */
    issueTypeUUID?: boolean
    /** 负责人 */
    assign?: boolean
    /** 任务描述 */
    descRich?: boolean
    /** 所属迭代 */
    sprintUUID?: boolean
  }
  /** 用户点击确认，且不再继续创建下一个时的回调 */
  onOk?: (result: TaskCreatedResult) => void
  /** 点击取消按钮时的回调 */
  onCancel?: () => void
  /** 工作项被创建之后的回调 */
  onTaskCreated?: (task: TaskCreatedResult['task']) => void
  /** 其他属性 */
  extra?: Record<string, any>
}

createSubTask​

唤起新建子工作项的弹窗

要求​
ONES@ones-op/bridge
v3.13.14v0.2.0
参数​
参数说明类型必填默认值
modalProps弹窗的配置CreateSubTaskModalProps是
类型​
interface CreateSubTaskModalProps extends CreateTaskModalProps {
  /** 父工作项详情 */
  parentTask: {
    /** 父工作项的ID */
    uuid: string
    /** 父工作项所属的项目ID */
    project_uuid: string
    [key: string]: any
  }
}
注意

parentTask中必须包含uuid和project_uuid，否则可能导致子工作项创建不成功。你可以直接传入工作项详情接口返回的数据

createRelatedTask​

唤起新建关联工作项的弹窗

要求​
ONES@ones-op/bridge
v3.13.14v0.2.0
参数​
参数说明类型必填默认值
modalProps弹窗的配置CreateRelatedTaskModalProps是
类型​
interface CreateRelatedTaskModalProps extends Omit<CreateTaskModalProps, 'defaultValue'> {
  /** 表单默认值 */
  defaultValue?: CreateTaskModalProps['defaultValue'] & {
    /** 关联关系 */
    relatedTypeValue?: string
  }
  /** 要关联的工作项 */
  relatedTask: {
    /** 关联工作项的ID */
    uuid: string
    /** 关联工作项的类型 */
    issue_type_uuid: string
    [key: string]: any
  }
  /** 工作项被关联之后的回调 */
  onRelatedSuccess?: (data: Record<string, any>) => void
}
注意

relatedTask中必须包含uuid和issue_type_uuid，否则可能导致工作项关联不成功。你可以直接传入工作项详情接口返回的数据

refreshList​

通过调用refreshList方法，你可以手动触发标品中列表的刷新

要求​

ONES@ones-op/bridge
v3.13.24+v0.3.0+

参数​

参数说明类型必填默认值
listName列表的名字ListName是

类型​

type ListName =
  | 'memberList'
  | 'codeRepositoryList'
  | 'planTestcaseList'
  | 'testPlanList'
  | 'requirementTraceabilityList'
memberList​
  • 可用：v0.3.0+
  • ONES 要求：v3.11.15+

触发成员列表刷新（支持标品中所有成员列表）。

codeRepositoryList​
  • 可用：v0.3.0+
  • ONES 要求：v3.13.9+

代码仓管理列表页刷新

planTestcaseList​
  • 可用：v0.3.0+
  • ONES 要求：v3.13.24+

测试管理 / 测试计划 / 用例列表刷新

testPlanList​
  • 可用: v0.3.1+
  • ONES 要求: v3.13.35+

测试管理 / 测试计划 / 测试计划列表刷新

requirementTraceabilityList​
  • 可用：v0.3.0+
  • ONES 要求：v3.13.24+

测试管理 / 测试计划 / 需求跟踪列表刷新

invokeProgressManager​

通过调用invokeProgressManager方法，可以激活全局进度管理器。

要求​

ONES@ones-op/bridge
v3.13.24+v0.3.0+

lifecycle​

通过调用 lifecycle 实例的方法，插件可以操作自身的部分生命周期。

要求​

@ones-op/bridge
v0.9.0+

destroy​

插件可以通过调用 lifecycle.destroy 方法，来销毁自身。

警告

这通常在手动激活的插件模块中使用，如果你使用的插槽并无特别说明，一般情况下不需要调用该方法。

  • ONES 要求：v3.11.40+

onDestroy​

在销毁插件之前被调用。

要求​
ONES@ones-op/bridge
v3.13.58+v0.6.1+
参数​
参数说明类型必填默认值
callback当前回调会在销毁插件之前被调用。CallbackFn是
类型​
function onDestroy(callback: () => void): void

dashboardCard​

通过 dashboardCard 模块提供的 hook，你可以完成 仪表盘卡片 的相关功能

警告

该模块仅在 仪表盘卡片 的插件模块中可用

要求​

ONES@ones-op/bridge
v3.13.50+v0.6.0+

useOnSubmitConfig​

在卡片配置弹窗中（scene 为 setting）调用 useOnSubmitConfig 方法，可以在用户点击保存配置时，触发你传入的回调。

参数​
参数说明类型必填默认值
callback用户点击弹窗上的保存按钮时的回调CallbackFn是
类型​
type CallbackFn = <T = any>(submit: (config: T) => void) => void | Promise<void>

function useOnSubmitConfig<T>(callback: CallbackFn<T>): void

useCustomMenus​

在卡片渲染区域中（scene 为 render）调用 useCustomMenus 方法，可以给卡片挂载自定义菜单

参数​
参数说明类型必填默认值
customMenus自定义菜单CustomMenuItem[]是
类型​
type CustomMenuItem = {
  key: string; // 菜单的唯一标识
  label: string; // 菜单标题
  onClick: () => void ｜ Promise<void>; // 点击菜单的回调
};

function useCustomMenus(customMenus: CustomMenuItem[]): void;

OPProvider​

当插件调用函数 lifecycle.destroy 时，OPProvider 将卸载其子组件。当插件在 YAML 中配置了属性 enableDelayDestroy 并设置为 true 来延迟销毁组件时，这个 provider 会将自己的子组件卸载。

要求​

ONES@ones-op/bridge
v3.13.58+v0.6.1+
