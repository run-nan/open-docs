# 下载与安装

开放平台诊断工具下载与安装。

## 命令行界面工具

### 在线安装

在联网环境下，我们更推荐你使用 Node.js 在线安装，在线安装可无需考虑系统架构，且更易于后续的升级。

安装最新版本：

```bash npm2yarn
npm install -g @ones/diagtools-cli
```

安装特定版本：

```bash npm2yarn
npm install -g @ones/diagtools-cli@[version]
```

使用：

```bash
diagtools --help
```

### 离线安装

对于内网环境，以及不想安装 Node.js 的用户，我们推荐你下载离线压缩包，里面包含了 CLI 工具所需要的所有依赖，你可以直接解压使用。

下载离线包后，你需要使用 tar 解压。

```bash
tar -zxvf [name].tar.gz
```

使用方式：

```bash
./diagtools/bin/diagtools --help
```

或者将解压后的 `diagtools/bin/diagtools` 添加到你系统的 bin 目录，全局使用：

```bash
diagtools --help
```

## 桌面端工具

如果你的系统是 MacOS 且版本是 12 或以上，首次安装时你可能会遇到：

<Image
  style={{ maxWidth: 250 }}
  img={require('./images/macos-big-sur-alert-unverified-developer.png')}
/>

你需要前往「系统偏好设置」里的「安全性与隐私」。点按「通用」面板中的「仍要打开」按钮，以确认你打算打开或安装这个 App。

然后再次执行安装动作即可完成安装。
