## 插槽

**模块路径：** 配置中心 / 新模块

**个数限制：** 一个插件中最多可声明 **10 **个该插槽，系统中最多同时存在 **100 **个该插槽

## 使用

```yaml
modules:
  - id: ones-settings-team-MSmf
    title: Team settings
    moduleType: ones:settings:team
    modules:
      - id: ones-settings-team-wd2A
        title: Team info
        modules:
          - id: ones-settings-team-MWB6
            title: Team info
            entry: >-
              modules/ones-settings-team-MSmf/ones-settings-team-wd2A/ones-settings-team-MWB6/index.html
```

**支持的配置项：**

- [disabled](../../../../reference/config/plugin.yaml#disabled)
- [modules（必填）](../../../../reference/config/plugin.yaml#subModule)

### 插槽层级

配置中心的每一项配置都是由三层组成。下面是「测试管理配置」的层级结构。

<Image style={{ maxWidth: '50%' }} img={require('./images/level.png')} />

部分配置界面上可能没有展示中间一层的层级，比如：团队配置，但是插槽会新增一个默认分组当作第二层级，你可以替换这个默认分组（ones:settings:team:default），
也支持在中间层级追加一个新的分组（ones:settings:team:new）。
