## 概述

### 能力描述

开放平台提供一种自定义工作项快捷操作按钮的能力，使用后，可以在全局视图配置中添加自定义快捷操作按钮，支持自定义按钮功能及顺序调整，应用于任意工作项类型后，即可在对应工作项中使用自定义的快捷操作，通过使用该能力能够实现更为个性化的功能，例如一下场景：

1. 当你想自定义一种工作项快捷操作，如导出评论、关联相关代码仓、推送微信公众号等非预设快捷操作。

### 能力表现

#### 添加插件快捷操作

当启用使用该能力的插件后，可以在工作项全局视图配置页面添加自定义的“插件快捷操作”。

<Image img={require('./plugin-quick-action-add.jpg')} />

#### 调整插件快捷操作位置

添加“插件快捷操作”成功后，可以在“概要信息卡片配置”查看到添加的“插件快捷操作”，可以像其他快捷操作一样调整位置和是否外部显示。

<Image img={actionModify} />

#### 发布全局视图

发布全局视图后，可以在工作项详情表单右上角，查看到添加的“插件快捷操作”，而其对应的功能则由插件定义。

<Image img={actionShow} />

## 能力使用

### 前置依赖

进入 /web 目录，通过以下命令安装 `@ones-op/store`

```
  npm install @ones-op/store
```

### 使用步骤

#### 添加插槽

执行以下命令，选中 `ones:global:modal`

```
  npx op add module
```

#### 添加能力

执行以下命令，选中 `layout-custom-quick-action@1.0.0`

```
  npx op add ability
```

#### 能力声明

添加完插槽和能力后，便可在 /config/plugin.yaml 文件中看到以下配置文件。

```yaml
service:
  app_id: MS6wAcF0
  ...
modules:
  - id: ones-global-modal-F3qL
    entry: modules/ones-global-modal-F3qL/index.html
    moduleType: ones:global:modal
    title: 插件快捷操作 # 按钮名称
    icon: logo.svg # 快捷操作按钮图标，取自项目下 web/public/logo.svg，默认使用@ones-design/icon的plugin图标，建议使用纯色svg图标与原快捷操作图标保持一致
    preload: true # 必须项，需手动填写
    manual: true # 必须项，需手动填写
abilities:
  - id: QayeaBmz
    name: 自定义视图快捷操作
    version: 1.0.0
    abilityType: LayoutCustomQuickAction
    relateModule:
      quickActionMenu: ones-project-settings-issueType-F3HS # 初始值为空，此处需要手动填写补充对应的插槽moduleID

```

:::caution 注意

1. 自定义视图快捷操作能力参数 `quickActionMenu` 值，为对应的自定义插槽 `id`，需要 `手动` 填写；
2. 自定义插槽的必须是 [ones:global:modal](../../api/slot/global.md#onesglobalmodal) 类型，自定义视图快捷操作的名称为关联插槽的 `title` 参数值；
3. 一个能力只能关联一个插槽，不允许多个能力关联同一个插槽。
4. `modules` 中的 `preload` 和 `manual` 属性需要 `手动` 配置成 `true`，否则按钮功能会自动激活，引起意料之外的问题。

:::

#### 自定义快捷操作按钮行为

插件侧前端示例

##### 弹窗类型

```tsx
import React, { useEffect } from 'react'
import ReactDOM from 'react-dom'
import { ConfigProvider, Modal } from '@ones-design/core'
import { useTaskInfo, destroyCurrentModule } from '@ones-op/store'
import './index.css'

function App() {
  // 当前工作项uuid
  const { uuid } = useTaskInfo()

  return (
    <>
      <Modal
        onCancel={function noRefCheck() {
          // 取消时，销毁弹窗
          destroyCurrentModule()
        }}
        onOk={function noRefCheck() {
          // 点击确定，销毁弹窗
          destroyCurrentModule()
        }}
        title="自定义快捷操作"
        closable
        visible
        width={520}
      >
        自定义快捷操作
      </Modal>
    </>
  )
}

ReactDOM.render(
  <ConfigProvider>
    <App />
  </ConfigProvider>,
  document.getElementById('ones-mf-root')
)
```

##### 无界面纯逻辑类型

```tsx
  import React, { useEffect } from 'react'
  import ReactDOM from 'react-dom'
  import { ConfigProvider } from '@ones-design/core'
  import { useTaskInfo, destroyCurrentModule } from '@ones-op/store'
  import './index.css'

  function App() {
    // 当前工作项uuid
    const { uuid } = useTaskInfo()

    useEffect(() => {
      // 执行相关操作逻辑
      ...
      // 最后销毁该插槽，否则无法再次触发
      destroyCurrentModule()
    })

    return <></>
  }

  ReactDOM.render(<ConfigProvider><App/></ConfigProvider>, document.getElementById('ones-mf-root'))
```

## 示例

### 场景一：自定义快捷操作按钮 「推送微信公众号」 用于推送关键信息到微信公众号

#### 场景描述

用户希望在工作项增加自定义快捷操作「推送微信公众号」，实现把工作项的状态情况能同步推送到微信公众号上的需求。

#### 实现思路

使用自定义工作项快捷操作能力，在前端插件中利用 `useTaskInfo` 获取到当前操作工作项的 `uuid`，利用 ONES 接口获取到当前工作项详细信息，拿到信息后调用微信公众号推送接口，从而实现「推送微信公众号」的功能。

#### 插件开发

前端侧插件 demo

```tsx
import React, { useEffect } from 'react'
import ReactDOM from 'react-dom'
import { ConfigProvider, toast } from '@ones-design/core';
import { useTaskInfo, destroyCurrentModule } from '@ones-op/store'
import { OPFetch } from '@ones-op/fetch';
import './index.css'

function App() {
  const { uuid } = useTaskInfo()
  const teamUUID = localStorage.getItem('teamUUID')

  // 获取工作项信息
  const getTaskInfo = () => {
    return OPFetch(`/project/P8091/api/project/team/${teamUUID}/task/${uuid}/info`)
  }

  // 推送到微信公众号
  const wxPublicAccountPush = (taskInfo) => {
    try {
      // 调用微信推送接口
      ...
      toast.success('推送成功')
    } catch (e) {
      toast.error('推送失败')
    }
  }

  const initialDeal = async () => {
    try {
      const { data } = await getTaskInfo()
      wxPublicAccountPush(data)
    } catch (e) {
      toast.error(e)
    }
  }

  useEffect(() => {
    initialDeal()
    // 销毁当前插槽
    setTimeout(() => {
      destroyCurrentModule()
    }, 0);
  }, [])

  return <></>
}
ReactDOM.render(<ConfigProvider><App/></ConfigProvider>, document.getElementById('ones-mf-root'))
```

## FAQ

### 自定义视图快捷操作能力与插槽如何绑定？

`plugin.yaml` 文件的配置参数 `quickActionMenu`，用于绑定自定义视图快捷操作能力与前端插槽关联关系。

### 如果我需要添加多个插件快捷操作怎么办？

自定义视图快捷操作能力与插槽是一对一的关系，同时基于功能独立性的原则，建议开发者一个插件只定义一个自定义视图快捷操作能力。

### 插件启用、停用和卸载，对工作项全局视图的影响？

1. 插件启用：

- 所有的工作项全局视图，可添加“插件快捷操作”；
- 已配置到全局视图的“插件快捷操作”，再次启用后将重新显示。

2. 插件停用：

- 所有的工作项全局视图，“插件快捷操作”不可添加；
- 已配置到全局视图的“插件快捷操作”，不再显示；
- 配置了“插件快捷操作”全局视图的草稿，此时数据非法不可编辑，可以取消发布退出重新进入。

3. 插件卸载：

- 所有的工作项全局视图，“插件快捷操作”不可添加；
- 之前配置的“插件快捷操作”会被删除，插件重新安装将无法找回，需重新配置;
- 配置了“插件快捷操作”全局视图的草稿，此时数据非法不可编辑，可以取消发布退出重新进入。
